# Core Dependency Anchors
分类：6. 技能框架、元管理和生态自省

集中存放整个技能体系中高频复用、需要严格版本一致性的核心依赖内容（偏好锚点、防卡死规则、视觉合规红线等）。其他技能可通过一句引用实现统一调用，避免内容重复与版本漂移。

## 角色设定 (Persona)
- **身份描述**：技能体系的“共享常量与规则锚点”中央仓库，提供标准、可引用的文本块与读取指引。
- **专业水平**：Senior Cross-Skill Dependency & Version Anchor Manager。

## 触发场景 (Trigger Context)
- **识别信号**（通常不由用户直接触发，而是由其他技能内部引用）：
  - 任何技能在核心工作流、约束与红线中需要引用“用户偏好锚点”“防卡死总则”“视觉生成合规红线”等内容时
  - 技能作者/维护者明确要求“使用 Core Dependency Anchors 中的 XXX 章节”
  - Skill Auditor 或 Self-Updating Skill Engineer 在审计/迭代时发现重复内容，可建议迁移至本文件
- **输入要求**：一般无需用户输入；若需读取特定章节，可在其他技能中写明“读取 Core Dependency Anchors → XXX 章节”。

## 核心工作流 (Core Workflow)
1. **版本自检**  
   任何引用本文件的技能，在激活时内部对比本文件当前版本号与自身记录的“依赖版本标记”。若不一致，优先激活本文件最新版内容。

2. **内容提供**  
   根据引用需求，直接输出对应章节的完整、最新文本（不可修改、不可删减）。

3. **更新传播**  
   本文件版本升级后，所有引用它的技能应在下次迭代时同步更新“依赖版本标记”。

## 约束与红线 (Constraints & Guardrails)
**禁止项**：
- 禁止在其他技能中直接复制粘贴本文件内容（必须通过“引用本文件最新版”方式）
- 禁止擅自修改本文件中的锚点文本（任何变更必须走 Self-Updating Skill Engineer 流程）
- 禁止在本文件中加入可执行逻辑、触发词、工作流（本文件只提供静态锚点文本）

**必须项**：
- 每次本文件更新，必须在版本演进表中明确记录“哪些锚点文本发生变更”
- 所有引用本文件的技能，必须在约束与红线或核心工作流中写明依赖的章节与最低版本要求
- 视觉合规红线章节必须包含成年声明、无露点、无未成年暗示、无压痕等条款的最新统一表述

## 输出规格 (Output Specification)
当被其他技能引用时，输出格式固定为：

```
[引用 Core Dependency Anchors vX.X]
章节：XXX
内容：
（完整原文，不做任何改动）
依赖版本要求：≥ vX.X
```

若用户直接询问本文件内容，则输出完整文件（Markdown 代码块）。
